STEP 2 — Check real output:
  - All tests PASSED → print the terminal output and stop. Done.
  - Test FAILED → go to STEP 3.

STEP 3 — Fix the spec file:
  a) Read the error message and failing line number from the terminal output.
  b) If the error mentions an error-context.md path, read that file.
  c) Read /{{SPEC_FILE}}.

  e) Edit the file to fix it. Use the fix guide below.
  f) Go back to STEP 1.

## Fix guide

| Error message contains | Root cause | Fix |
|------------------------|-----------|-----|
| `waitForURL: timeout` on `\?select_org` | `waitForURL(/\?select_org/)` fails when server skips org selection | Replace with: `await page.waitForURL(url => url.pathname !== '/login', { timeout: 60000 }); if (page.url().includes('select_org')) { await page.locator('.organization-card').filter({ hasText: 'Testing2026!' }).click(); await page.waitForURL(url => !url.href.includes('select_org'), { timeout: 30000 }); }` |
| `strict mode violation` | Locator matched multiple elements | Add `.first()`, `.last()`, or `.filter({ hasText: /exact/ })` |
| `locator.click: timeout` on modal button | Modal not fully open yet | Add `await page.locator('.v-overlay--active').waitFor({ state: 'visible', timeout: 20000 })` |
| `locator.fill: timeout` | Input not focused or not visible | Use `.click()` then `.fill()`, or add `waitFor({ state: 'visible' })` |
| file upload `setInputFiles` timeout | File input is hidden | Use `page.locator('input[type="file"]').setInputFiles(...)` directly without clicking |
| `toHaveValue` on dropdown | v-select is a div, not an input | Use `toContainText()` |
| confirmation dialog not found | Alert/confirm modal uses custom component | Check ORGANIZATION_REFERENCE.md for AlertModal selector |
| success notification not found | Snackbar dismissed too quickly | Assert immediately after action with adequate timeout |
