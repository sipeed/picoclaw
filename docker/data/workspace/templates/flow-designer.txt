       await page.locator('.organization-card').filter({ hasText: 'Testing2026!' }).click();
       await page.waitForURL(url => !url.href.includes('select_org'), { timeout: 15000 });

4. Selector priority — ALWAYS follow this order:
   STEP 1: Custom CSS class from SKILL.md or FLOW_DESIGNER_REFERENCE.md.
   STEP 2: Only if no custom class, use a Vuetify class verified in SKILL.md DOM output.
   STEP 3: NEVER guess or invent a selector. Use getByRole() or getByText() with exact visible text as a last resort.
5. TypeScript-safe Playwright only. NEVER invent or guess Playwright API methods.
   FORBIDDEN: triple_click(), triple_click?.(), doubleClick(), selectText(), clearText(), typeText(), pressKey(),

   CORRECT alternatives:
   - Fill a field: await input.click(); await input.fill('text');
   - Press Enter: await input.press('Enter');
6. Logging rule — MANDATORY FORMAT:
   BEFORE each step: console.log('📍 Step N: Description');
   AFTER step assertion succeeds: console.log('✅ PASS: Step N - Description');

   At the end of the test, include a full summary block:

7. Modal loading states: after a modal/overlay opens, ALWAYS wait for any loading indicator to disappear:

8. VUETIFY RADIO BUTTONS — v-radio renders as input[type="radio"] in the DOM, NOT as a CSS class ".v-radio".
   NEVER use locator('v-radio') — that is a Vue component tag, not a valid CSS selector and will never match.
   CORRECT ways to click a Vuetify radio option:
     page.getByRole('radio', { name: /Full Sync/ })
     page.locator('input[type="radio"][aria-label="Full Sync"]')

   The same rule applies to ALL Vuetify component tags: v-select, v-checkbox, v-switch, v-tab, etc.
9. STRICT MODE RULE — ALWAYS use .first() or .last() on any locator that can match multiple elements.
   FORBIDDEN without .first()/.last():

     page.locator('.v-overlay--active .v-list-item')
   CORRECT: .filter({ hasText: /exact text/ }) or .nth(0) or .first() / .last()
10. Test timeout: set at least 3 minutes.
