## 📦 انسٹال

### پرے کمپائل شدہ بائنری کے ساتھ انسٹال کریں

اپنے پلیٹ فارم کے لیے فرم ویئر [ریلیز](https://github.com/sipeed/picoclaw/releases) صفحہ سے ڈاؤن لوڈ کریں۔

### ماخذ سے انسٹال کریں (تازہ ترین خصوصیات، ترقی کے لیے سفارش کی گئی)

```bash
git clone https://github.com/sipeed/picoclaw.git

cd picoclaw
make deps

# تعمیر کریں، انسٹال کرنے کی ضرورت نہیں
make build

# متعدد پلیٹ فارمز کے لیے تعمیر کریں
make build-all

# تعمیر اور انسٹال کریں
make install
```

## 🐳 Docker Compose

آپ Docker Compose کا استعمال کرتے ہوئے PicoClaw کو چلا سکتے ہیں بغیر کسی چیز کو مقامی طور پر انسٹال کیے۔

```bash
# 1. اس ریپو کو کلون کریں
git clone https://github.com/sipeed/picoclaw.git
cd picoclaw

# 2. اپنی API کلیدیں سیٹ کریں
cp config/config.example.json config/config.json
vim config/config.json      # DISCORD_BOT_TOKEN، API کلیدیں، وغیرہ سیٹ کریں

# 3. تعمیر اور شروع کریں
docker compose --profile gateway up -d

> [!TIP]
> **Docker صارفین**: بطور ڈیفالٹ، Gateway `127.0.0.1` پر سنتا ہے جو میزبان سے قابل رسائی نہیں ہے۔ اگر آپ کو ہیلتھ اینڈ پوائنٹس تک رسائی حاصل کرنی ہے یا پورٹس میں نمایاں کرنا ہے، تو اپنی ماحول میں `PICOCLAW_GATEWAY_HOST=0.0.0.0` سیٹ کریں یا `config.json` کو اپ ڈیٹ کریں۔

# 4. لاگز چیک کریں
docker compose logs -f picoclaw-gateway

# 5. رکیں
docker compose --profile gateway down
```

### ایجنٹ موڈ (ایک بار کا)

```bash
# سوال پوچھیں
docker compose run --rm picoclaw-agent -m "2+2 کیا ہے؟"

# انٹرایکٹو موڈ
docker compose run --rm picoclaw-agent
```

### دوبارہ تعمیر کریں

```bash
docker compose --profile gateway build --no-cache
docker compose --profile gateway up -d
```

### 🚀 فوری شروعات

> [!TIP]
> اپنی API کلید `~/.picoclaw/config.json` میں سیٹ کریں۔
> API کلیدیں حاصل کریں: [OpenRouter](https://openrouter.ai/keys) (LLM) · [Zhipu](https://open.bigmodel.cn/usercenter/proj-mgmt/apikeys) (LLM)
> ویب سرچ **اختیاری** ہے - مفت [Tavily API](https://tavily.com) حاصل کریں (1000 مفت سوالات/ماہ) یا [Brave Search API](https://brave.com/search/api) (2000 مفت سوالات/ماہ) یا بلٹ ان خودکار fallback استعمال کریں۔

**1. شروع کریں**

```bash
picoclaw onboard
```

**2. ترتیب دیں** (`~/.picoclaw/config.json`)

```json
{
  "agents": {
    "defaults": {
      "workspace": "~/.picoclaw/workspace",
      "model_name": "gpt4",
      "max_tokens": 8192,
      "temperature": 0.7,
      "max_tool_iterations": 20
    }
  },
  "model_list": [
    {
      "model_name": "gpt4",
      "model": "openai/gpt-5.2",
      "api_key": "your-api-key",
      "request_timeout": 300
    },
    {
      "model_name": "claude-sonnet-4.6",
      "model": "anthropic/claude-sonnet-4.6",
      "api_key": "your-anthropic-key"
    }
  ],
  "tools": {
    "web": {
      "brave": {
        "enabled": false,
        "api_key": "YOUR_BRAVE_API_KEY",
        "max_results": 5
      },
      "tavily": {
        "enabled": false,
        "api_key": "YOUR_TAVILY_API_KEY",
        "max_results": 5
      },
      "duckduckgo": {
        "enabled": true,
        "max_results": 5
      }
    }
  }
}
```

> **نیا**: `model_list` ترتیب کی شکل صفر کوڈ فراہم کنندہ اضافے کی اجازت دیتی ہے۔ تفصیلات کے لیے [ماڈل ترتیب](#model-configuration-model_list) دیکھیں۔
> `request_timeout` اختیاری ہے اور سیکنڈ میں ہے۔ اگر حذف کیا جائے یا `<= 0` پر سیٹ کیا جائے، تو PicoClaw ڈیفالٹ ٹائم آؤٹ استعمال کرتا ہے (120s)۔

**3. API کلیدیں حاصل کریں**

* **LLM فراہم کنندہ**: [OpenRouter](https://openrouter.ai/keys) · [Zhipu](https://open.bigmodel.cn/usercenter/proj-mgmt/apikeys) · [Anthropic](https://console.anthropic.com) · [OpenAI](https://platform.openai.com) · [Gemini](https://aistudio.google.com/api-keys)
* **ویب سرچ** (اختیاری): [Tavily](https://tavily.com) - AI ایجنٹس کے لیے بہتری (1000 درخواستیں/ماہ) · [Brave Search](https://brave.com/search/api) - مفت سطح دستیاب (2000 درخواستیں/ماہ)

> **نوٹ**: مکمل ترتیب ٹیمپلیٹ کے لیے `config.example.json` دیکھیں۔

**4. بات چیت کریں**

```bash
picoclaw agent -m "2+2 کیا ہے؟"
```

بس! آپ کے پاس 2 منٹ میں کام کرنے والا AI معاون ہے۔

<details>
<summary><b>Discord</b></summary>

**1. بوٹ بنائیں**

* <https://discord.com/developers/applications> پر جائیں
* ایک ایپلیکیشن بنائیں → بوٹ → بوٹ شامل کریں
* بوٹ ٹوکن کاپی کریں

**2. intents فعال کریں**

* بوٹ کی ترتیبات میں، **MESSAGE CONTENT INTENT** فعال کریں
* (اختیاری) **SERVER MEMBERS INTENT** فعال کریں اگر آپ ممبر ڈیٹا کی بنیاد پر اجازت کی فہرستوں کو استعمال کرنا چاہتے ہیں

**3. اپنی صارف ID حاصل کریں**
* Discord ترتیبات → Advanced → **Developer Mode** فعال کریں
* اپنے اوتار پر دائیں کلک کریں → **Copy User ID**

**4. ترتیب دیں**

```json
{
  "channels": {
    "discord": {
      "enabled": true,
      "token": "YOUR_BOT_TOKEN",
      "allow_from": ["YOUR_USER_ID"],
      "mention_only": false
    }
  }
}
```

**5. بوٹ کو دعوت دیں**

* OAuth2 → URL Generator
* Scopes: `bot`
* بوٹ اجازتیں: `پیغامات بھیجیں`، `پیغام کی تاریخ پڑھیں`

## ⚙️ ترتیب

ترتیب فائل: `~/.picoclaw/config.json`

### Workspace کا منصوبہ

PicoClaw آپ کے ترتیب شدہ workspace میں ڈیٹا ذخیرہ کرتا ہے (ڈیفالٹ: `~/.picoclaw/workspace`):

```
~/.picoclaw/workspace/
├── sessions/          # گفتگو کے سیشنز اور تاریخ
├── memory/           # طویل مدتی میموری (MEMORY.md)
├── state/            # مستقل حالت (آخری چینل، وغیرہ)
├── cron/             # شیڈول شدہ کاموں کی ڈیٹا بیس
├── skills/           # کسٹم مہارتیں
├── AGENTS.md         # ایجنٹ کے رویے کی گائیڈ
├── HEARTBEAT.md      # دورانیہ کام کے اشارے (ہر 30 منٹ چیک کیا جاتا ہے)
├── IDENTITY.md       # ایجنٹ کی شناخت
├── SOUL.md           # ایجنٹ کی روح
├── TOOLS.md          # ٹول کی تفصیلات
└── USER.md           # صارف کی ترجیحات
```

### 🔒 سیکیورٹی Sandbox

PicoClaw بطور ڈیفالٹ ایک سینڈ بکس شدہ ماحول میں چلتا ہے۔ ایجنٹ صرف ترتیب شدہ workspace کے اندر فائلوں تک رسائی حاصل کر سکتا ہے اور کمانڈز انجام دے سکتا ہے۔

#### ڈیفالٹ ترتیب

```json
{
  "agents": {
    "defaults": {
      "workspace": "~/.picoclaw/workspace",
      "restrict_to_workspace": true
    }
  }
}
```

| آپشن                  | ڈیفالٹ                 | تفصیل                               |
| ----------------------- | ----------------------- | ----------------------------------------- |
| `workspace`             | `~/.picoclaw/workspace` | ایجنٹ کے لیے کام کرنے والی ڈائریکٹری           |
| `restrict_to_workspace` | `true`                  | فائل/کمانڈ رسائی کو workspace تک محدود کریں |

#### حفاظت سے محفوظ ٹولز

جب `restrict_to_workspace: true` ہو، تو درج ذیل ٹولز sandbox میں ہوتے ہیں:

| ٹول          | فنکشن         | پابندی                            |
| ------------- | ---------------- | -------------------------------------- |
| `read_file`   | فائلیں پڑھیں       | صرف workspace کے اندر فائلیں            |
| `write_file`  | فائلیں لکھیں      | صرف workspace کے اندر فائلیں            |
| `list_dir`    | ڈائریکٹریز درج کریں | صرف workspace کے اندر ڈائریکٹریز      |
| `edit_file`   | فائلیں ترمیم کریں       | صرف workspace کے اندر فائلیں            |
| `append_file` | فائلوں میں شامل کریں  | صرف workspace کے اندر فائلیں            |
| `exec`        | کمانڈز انجام دیں | کمانڈ راستے workspace کے اندر ہونے چاہیں |

#### اضافی Exec حفاظت

یہاں تک کہ `restrict_to_workspace: false` کے ساتھ، `exec` ٹول ان خطرناک کمانڈز کو روکتا ہے:

* `rm -rf`، `del /f`، `rmdir /s` — بڑے پیمانے پر حذف
* `format`، `mkfs`، `diskpart` — ڈسک فارمیٹنگ
* `dd if=` — ڈسک امیجنگ
* `/dev/sd[a-z]` میں لکھنا — براہ راست ڈسک لکھنا
* `shutdown`، `reboot`، `poweroff` — سسٹم شٹ ڈاؤن
* Fork bomb `:(){ :|:& };:`

#### خرابی کی مثالیں

```
[ERROR] tool: ٹول کی انجام دہی ناکام ہو گئی
{tool=exec, error=کمانڈ سیفٹی گارڈ کے ذریعے بلاک کیا گیا (کام کری ڈائریکٹری کے باہر راستہ)}
```

```
[ERROR] tool: ٹول کی انجام دہی ناکام ہو گئی
{tool=exec, error=کمانڈ سیفٹی گارڈ کے ذریعے بلاک کیا گیا (خطرناک پیٹرن دریافت کیا گیا)}
```

#### پابندیوں کو غیر فعال کریں (سیکیورٹی کا خطرہ)

اگر آپ کو ایجنٹ کو workspace کے باہر راستوں تک رسائی حاصل کرنے کی ضرورت ہے:

**طریقہ 1: ترتیب فائل**

```json
{
  "agents": {
    "defaults": {
      "restrict_to_workspace": false
    }
  }
}
```

**طریقہ 2: ماحول متغیر**

```bash
export PICOCLAW_AGENTS_DEFAULTS_RESTRICT_TO_WORKSPACE=false
```

> ⚠️ **انتباہ**: اس پابندی کو غیر فعال کرنا ایجنٹ کو آپ کے نظام پر کسی بھی راستے تک رسائی حاصل کرنے کی اجازت دیتا ہے۔ صرف کنٹرول شدہ ماحول میں احتیاط سے استعمال کریں۔

#### سیکیورٹی کی حدود میں مطابقت

`restrict_to_workspace` ترتیب تمام execution راستوں میں مطابقت سے لاگو ہوتی ہے:

| Execution راستہ   | سیکیورٹی کی حدود            |
| ---------------- | ---------------------------- |
| مین ایجنٹ       | `restrict_to_workspace` ✅   |
| Subagent / Spawn | ایک جیسی پابندی لیتا ہے ✅ |
| Heartbeat کام  | ایک جیسی پابندی لیتا ہے ✅ |

تمام راستے ایک جیسی workspace پابندی کا اشتراک کرتے ہیں — subagents یا شیڈول شدہ کاموں کے ذریعے سیکیورٹی کی حدود کو نقل کرنے کا کوئی طریقہ نہیں ہے۔

### Heartbeat (دورانیہ کام)

PicoClaw خود بخود دورانیہ کام انجام دے سکتا ہے۔ اپنے workspace میں ایک `HEARTBEAT.md` فائل بنائیں:

```markdown
# دورانیہ کام

- اہم پیغاموں کے لیے میری ای میل چیک کریں
- آنے والے واقعات کے لیے میرے کیلنڈر کو دیکھیں
- موسم کی پیشن گوئی چیک کریں
```

ایجنٹ ہر 30 منٹ میں (ترتیب دینے کے قابل) اس فائل کو پڑھے گا اور دستیاب ٹولز کا استعمال کرتے ہوئے کسی بھی کام کو انجام دے گا۔

#### Spawn کے ساتھ Async کام

طویل مدتی کاموں (ویب سرچ، API کالز) کے لیے، ایک **subagent** بنانے کے لیے `spawn` ٹول استعمال کریں:

```markdown
# دورانیہ کام

## تیز کام (براہ راست جواب دیں)

- موجودہ وقت رپورٹ کریں

## لمبے کام (async کے لیے spawn استعمال کریں)

- AI خبروں کے لیے ویب سرچ کریں اور خلاصہ کریں
- ای میل چیک کریں اور اہم پیغاموں کی رپورٹ کریں
```

**اہم رویے:**

| خصوصیت                 | تفصیل                                               |
| ----------------------- | --------------------------------------------------------- |
| **spawn**               | Async subagent بناتا ہے، heartbeat کو بلاک نہیں کرتا           |
| **آزاد سیاق** | Subagent کا اپنا سیاق ہے، کوئی سیشن تاریخ نہیں          |
| **پیغام ٹول**        | Subagent پیغام ٹول کے ذریعے صارف کے ساتھ براہ راست رابطہ کرتا ہے |
| **غیر مسدود**        | spawn کرنے کے بعد، heartbeat اگلے کام میں جاتا ہے          |

#### Subagent رابطے کی کارکردگی

```
Heartbeat ٹریگر
    ↓
ایجنٹ HEARTBEAT.md پڑھتا ہے
    ↓
طویل کام کے لیے: subagent spawn کریں
    ↓                           ↓
اگلے کام پر جائیں      Subagent آزادانہ طریقے سے کام کرتا ہے
    ↓                           ↓
تمام کاموں کے بعد            Subagent "message" ٹول استعمال کرتا ہے
    ↓                           ↓
HEARTBEAT_OK کا جواب      صارف براہ راست نتیجہ وصول کرتا ہے
```

Subagent کے پاس ٹولز (message، web_search، وغیرہ) تک رسائی ہے اور مین ایجنٹ سے گزرے بغیر صارف کے ساتھ آزادانہ طریقے سے رابطہ کر سکتا ہے۔

**ترتیب:**

```json
{
  "heartbeat": {
    "enabled": true,
    "interval": 30
  }
}
```

| آپشن     | ڈیفالٹ | تفصیل                        |
| ---------- | ------- | ---------------------------------- |
| `enabled`  | `true`  | Heartbeat کو فعال/غیر فعال کریں           |
| `interval` | `30`    | منٹوں میں چیک وقفہ (کم: 5) |

**ماحول متغیرات:**

* Heartbeat کو غیر فعال کرنے کے لیے `PICOCLAW_HEARTBEAT_ENABLED=false`
* وقفہ کو تبدیل کرنے کے لیے `PICOCLAW_HEARTBEAT_INTERVAL=60`

### ماڈل ترتیب (model_list)

> **کیا نیا ہے؟** PicoClaw اب ایک **ماڈل مرکوز** ترتیب کا طریقہ استعمال کرتا ہے۔ صرف `vendor/model` فارمیٹ مخصوص کریں (مثال کے طور پر، `zhipu/glm-4.7`) نئے فراہم کنندگان کو شامل کرنے کے لیے — **صفر کوڈ تبدیلیاں درکار ہیں!**

یہ ڈیزائن **ملٹی ایجنٹ سپورٹ** کو بھی فعال کرتا ہے لچکدار فراہم کنندہ کے انتخاب کے ساتھ:

- **مختلف ایجنٹ، مختلف فراہم کنندگان**: ہر ایجنٹ اپنے LLM فراہم کنندہ کو استعمال کر سکتا ہے
- **ماڈل fallbacks**: مضبوطی کے لیے بنیادی اور fallback ماڈلز ترتیب دیں
- **بوجھ میں توازن**: ایک سے زیادہ endpoints میں درخواستوں کو تقسیم کریں

### فراہم کنندہ کی تعمیر

PicoClaw پروٹوکول خاندان کے ذریعے فراہم کنندگان کو روٹ کرتا ہے:

- OpenAI مطابقت پروٹوکول: OpenRouter، OpenAI مطابقت gateways، Groq، Zhipu، اور vLLM-سٹائل endpoints۔
- Anthropic پروٹوکول: Claude-native API رویہ۔
- Codex/OAuth راستہ: OpenAI OAuth/ٹوکن تصدیق راستہ۔

یہ runtime کو ہلکا رکھتا ہے جبکہ نئے OpenAI-مطابقت backends کو زیادہ تر ترتیب کے آپریشن (`api_base` + `api_key`) بناتا ہے۔

| `picoclaw cron add ...`   | شیڈول شدہ کام شامل کریں           |

### شیڈول شدہ کام / یادیں

PicoClaw `cron` ٹول کے ذریعے شیڈول شدہ یادیں اور دہرائے جانے والے کام کو سپورٹ کرتا ہے:

* **ایک بار کی یادیں**: "مجھے 10 منٹ میں یاد دلائیں" → 10منٹ بعد ایک بار ٹریگر ہو
* **دہرائے جانے والے کام**: "مجھے ہر 2 گھنٹے میں یاد دلائیں" → ہر 2 گھنٹے میں ٹریگر ہو
* **Cron اظہار**: "مجھے روزانہ 9 بجے یاد دلائیں" → cron اظہار استعمال کرتا ہے

کام `~/.picoclaw/workspace/cron/` میں محفوظ ہیں اور خود بخود پروسیس ہوتے ہیں۔

## 🐛 مسائل کا حل

### ویب سرچ کہتا ہے "API کی ترتیب میں مسئلہ"

یہ عام ہے اگر آپ نے ابھی سرچ API کلید ترتیب نہیں دی ہے۔ PicoClaw دستی تلاش کے لیے مفید لنکس فراہم کرے گا۔

ویب سرچ کو فعال کرنے کے لیے:

1. **اپشن 1 (سفارش کی گئی)**: [https://brave.com/search/api](https://brave.com/search/api) پر مفت API کلید حاصل کریں (بہترین نتائج کے لیے ماہ میں 2000 مفت سوالات)۔
2. **اپشن 2 (کریڈٹ کارڈ نہیں)**: اگر آپ کے پاس کوئی کلید نہیں ہے، تو ہم خود بخود **DuckDuckGo** (کوئی کلید درکار نہیں) میں واپس ہو جاتے ہیں۔

Brave استعمال کر رہے ہیں تو `~/.picoclaw/config.json` میں کلید شامل کریں:

```json
{
  "tools": {
    "web": {
      "brave": {
        "enabled": false,
        "api_key": "YOUR_BRAVE_API_KEY",
        "max_results": 5
      },
      "duckduckgo": {
        "enabled": true,
        "max_results": 5
      }
    }
  }
}
```

### مواد فلٹرنگ خرابیاں حاصل کریں

کچھ فراہم کنندگان (جیسے Zhipu) میں مواد فلٹرنگ ہے۔ اپنی سوال کو دوبارہ بیان کرنے کی کوشش کریں یا ایک مختلف ماڈل استعمال کریں۔

### Telegram بوٹ کہتا ہے "تنازعہ: دوسرے getUpdates کے ذریعے ختم"

یہ اس وقت ہوتا ہے جب بوٹ کی دوسری مثال چل رہی ہو۔ یقینی بنائیں کہ ایک بار میں صرف ایک `picoclaw gateway` چل رہا ہے۔
