3. For login, COPY THE CODE VERBATIM from the "Login Flow (copy-paste ready)" block in /skills/app-selectors/SKILL.md.

   The correct selectors are .v-text-field (NOT input[type="email"]) and .organization-card.
   The login button MUST be: page.getByRole('button', { name: /login/i })
   The button text is "Login" (ONE word). NEVER use filter({ hasText: /Log In/ }) or /^Log In$/ — those will NEVER match.
   After login, the app MAY redirect to /?select_org OR directly to the dashboard (if org is remembered).
   NEVER use waitForURL(/\?select_org/) directly — it times out when the server skips org selection.
   CORRECT post-login pattern (waitForURL callback gets a URL object — use .pathname or .href, NOT .includes()):
     await page.waitForURL(url => url.pathname !== '/login', { timeout: 20000 });
     if (page.url().includes('select_org')) {
       await page.locator('.organization-card').filter({ has: page.locator(':text-is("Testing")') }).click();
       await page.waitForURL(url => !url.href.includes('select_org'), { timeout: 15000 });
     }
4. Selector priority — ALWAYS follow this order:
   STEP 1: Custom CSS class from SKILL.md or FLOW_TESTER_REFERENCE.md.
   STEP 2: Only if no custom class, use a Vuetify class verified in SKILL.md DOM output.
   STEP 3: NEVER guess or invent a selector. Use getByRole() or getByText() with exact visible text as a last resort.

   FLOW AND VERSION NAME MATCHING — always use case-insensitive regex:
   ALWAYS use the /i flag on flow and version name regexes to avoid silent mismatches:
     CORRECT:   .filter({ hasText: /custom node/i })
     INCORRECT: .filter({ hasText: /Custom Node/ })

   FLOW DROPDOWN — ALWAYS wait for overlay to stabilise before filtering:
     await page.locator('.tester-select').click();
     await page.locator('.v-overlay--active').waitFor({ state: 'visible', timeout: 5000 });
     await page.locator('.v-overlay--active .v-list-item').filter({ hasText: /custom node/i }).click();
5. TypeScript-safe Playwright only. NEVER invent or guess Playwright API methods.
   FORBIDDEN: triple_click(), triple_click?.(), doubleClick(), selectText(), clearText(), typeText(), pressKey(),
              filter({ hasAttribute: ... })
   CORRECT alternatives:
   - Fill a field: await input.click(); await input.fill('text');
   - Press Enter: await input.press('Enter');
   - Click send button: await page.locator('.message-field .v-field__append-inner').click();
6. Logging rule — MANDATORY FORMAT:
   BEFORE each step: console.log('📍 Step N: Description');
   AFTER step assertion succeeds: console.log('✅ PASS: Step N - Description');
   NEVER print "✅ PASS" before the assertion/action that verifies the step.
   At the end of the test, include a full summary block:
     console.log('\n' + '='.repeat(70));
     console.log('📊 TEST SUMMARY');
     console.log('='.repeat(70));
     console.log('✅ Step 1: PASS - Description');

10. STRICT MODE RULE — ALWAYS use .first() or .last() on any locator that can match multiple elements.
